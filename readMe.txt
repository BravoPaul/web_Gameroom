项目简介：
这个project 是类似于qq游戏大厅的一个web项目，目前上边只有3个游戏，分别是：中国象棋，五子棋，和法国的一个棋类游戏。在这个平台上可以像qq游戏一样实现俩个人对战。
项目构架：
项目使用了Gateway这个开源架构，Gateway是基于Workerman架构开发的。其实就是将WorkerMan的函数进行封装变得更加容易使用。WorkerMan是一款纯PHP开发的开源高性能的PHP socket 服务器框架。被广泛的用于手机app、移动通讯，微信小程序，手游服务端、网络游戏、PHP聊天室、硬件通讯、智能家居、车联网、物联网等领域的开发。反正他是一个非常牛逼的东西。如果想详细了解请打开链接：http://www.workerman.net/ 和 http://www.workerman.net/gatewaydoc-1.0/。
项目内容：
因为我们使用了Gateway 这个架构，所以就不用来管底层的通信问题，只需要运用其封装的函数来进行点对点通信或者广播通信。首先当有用户想链接游戏大厅时，他会请求服务器，服务器这时会记录刚刚链接的用户的ID并把它存储到用户列表里。然后我们需要为每个游戏开发一个客户端。在这个客户端里包括：棋盘，棋子，棋子怎么移动，是否游戏结束等等游戏相关的内容。而这些开发都可以用js来搞定。当我们完成客户端后，例如我需要在客户端调用move()函数来移动了一个棋子，当你再客户端移动以后，需要告诉其他玩家你移动了棋子，那么你就可以命名刚才的动作例如“movechess”，发送给服务器，当服务器接收到“movechess”这个关键字后，我们可以用send()这个GateWay封装好的函数来像具体的玩家发送信号，当玩家接收到信号后，那么他也调用move()方法来实现对方玩家棋子的移动。那么玩家就看到对手走了一步棋。
